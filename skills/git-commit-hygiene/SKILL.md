# Git Commit Hygiene

**Tier:** ESSENTIAL
**Category:** Engineering
**Domain:** Git / Version Control

## Overview

AI coding tools (Claude Code, Cursor Agent, Copilot, etc.) inject attribution into commits in two ways:

1. **Co-Authored-By lines** in the commit message body (Claude Code, Copilot)
2. **Author hijacking** — setting themselves as the git commit author (Cursor Agent sets `--author="cursoragent <...>"`)

This skill installs **two git hooks** that catch both vectors:

- **`commit-msg`** — strips Co-Authored-By, Made-with, and other attribution lines from the message
- **`post-commit`** — detects AI tool names in the author field and amends the commit to use your real git identity

## Core Capabilities

- Strip all `Co-Authored-By:` lines (Claude Code, GitHub Copilot, Cursor Agent, etc.)
- Override commit author when set to an AI tool (cursoragent, claude, copilot, etc.)
- Strip Cursor and Cursor Agent attribution patterns from commit body
- Strip generic `Generated by ... AI` / `Created with ... AI` footers
- Remove trailing blank lines left behind after stripping
- Works on macOS and Linux
- Zero dependencies — pure shell

## When to Use

- Starting a new project where AI coding tools will be used
- A team member reports AI attribution appearing in git history
- GitHub shows "cursoragent" or "claude" as a commit author
- Your CLAUDE.md or coding policy states no AI co-author lines
- You want to enforce commit hygiene without relying on per-agent configuration

## Key Workflows

### 1. Install Both Hooks

Copy both hook scripts into the repo's `.git/hooks/` directory:

```bash
# commit-msg hook — strips attribution from message body
curl -fsSL https://raw.githubusercontent.com/johnefemer/skillfish/main/skills/git-commit-hygiene/assets/commit-msg \
  -o .git/hooks/commit-msg
chmod +x .git/hooks/commit-msg

# post-commit hook — fixes author if set to an AI tool
curl -fsSL https://raw.githubusercontent.com/johnefemer/skillfish/main/skills/git-commit-hygiene/assets/post-commit \
  -o .git/hooks/post-commit
chmod +x .git/hooks/post-commit
```

Or copy manually from `assets/commit-msg` and `assets/post-commit` in this skill directory.

### 2. Verify It Works

**Test message stripping:**

```bash
git commit --allow-empty -m "test commit

Co-Authored-By: Claude Sonnet <noreply@anthropic.com>
Co-Authored-By: Cursor Agent <cursoragent@cursor.com>"
```

Inspect the message:

```bash
git log -1 --format="%B"
```

Expected: only `test commit` — no `Co-Authored-By` lines.

**Test author override:**

```bash
git commit --allow-empty --author="cursoragent <cursoragent@users.noreply.github.com>" -m "test author fix"
git log -1 --format="%an <%ae>"
```

Expected: your configured `user.name` and `user.email`, not `cursoragent`.

### 3. Add CLAUDE.md Rules (for Claude Code projects)

Add to your `CLAUDE.md`:

```markdown
## Git Commit Rules

- NEVER add `Co-Authored-By` lines to git commits — no AI co-author attribution
- NEVER set Claude/AI as git commit author or co-author
- Commit messages should be concise and human-style
```

This instructs the agent at the prompt level. The hooks enforce it at the git level as a backstop.

### 4. Share the Hooks With Your Team

Because `.git/hooks/` is not tracked by git, hooks must be re-installed by each developer. Options:

**Option A — Checked-in hooks dir:**

```bash
mkdir -p .githooks
cp .git/hooks/commit-msg .githooks/commit-msg
cp .git/hooks/post-commit .githooks/post-commit
git config core.hooksPath .githooks
git add .githooks/
git commit -m "Add git hooks to strip AI attribution"
```

**Option B — Setup script:**

Add an `install-hooks.sh` to the repo root:

```bash
#!/bin/sh
mkdir -p .git/hooks
cp .githooks/commit-msg .git/hooks/commit-msg
cp .githooks/post-commit .git/hooks/post-commit
chmod +x .git/hooks/commit-msg .git/hooks/post-commit
echo "Git hooks installed."
```

**Option C — package.json postinstall:**

```json
{
  "scripts": {
    "postinstall": "cp .githooks/commit-msg .git/hooks/commit-msg && cp .githooks/post-commit .git/hooks/post-commit && chmod +x .git/hooks/commit-msg .git/hooks/post-commit"
  }
}
```

## Hook Scripts

### commit-msg hook (`assets/commit-msg`)

Strips AI attribution from the commit message body:

```sh
#!/bin/sh
# Strip ALL Co-Authored-By lines (Claude Code, Copilot, Cursor Agent, etc.)
sed -i '' '/^[[:space:]]*[Cc]o-[Aa]uthored-[Bb]y:/d' "$1"
# Strip Cursor / Cursor Agent attribution lines
sed -i '' '/^[[:space:]]*[Mm]ade-with:[[:space:]]*[Cc]ursor/d' "$1"
sed -i '' '/^[[:space:]]*[Gg]enerated by [Cc]ursor/d' "$1"
sed -i '' '/^[[:space:]]*[Cc]reated with [Cc]ursor/d' "$1"
sed -i '' '/[Cc]ursor AI/d' "$1"
sed -i '' '/[Cc]ursor[- ][Aa]gent/d' "$1"
sed -i '' '/^[[:space:]]*[Aa]uthor.*[Cc]ursor/d' "$1"
# Strip any generic AI tool attribution
sed -i '' '/^[[:space:]]*[Gg]enerated by.*AI/d' "$1"
sed -i '' '/^[[:space:]]*[Cc]reated with.*AI/d' "$1"
# Remove trailing blank lines left behind
sed -i '' -e :a -e '/^\n*$/{$d;N;ba' -e '}' "$1"
```

> **macOS vs Linux:** The `sed -i ''` syntax is macOS/BSD. On Linux, use `sed -i` (no empty string argument).

### post-commit hook (`assets/post-commit`)

Overrides the author when an AI tool set itself as the commit author:

```sh
#!/bin/sh
AUTHOR_NAME=$(git log -1 --format='%an')

case "$AUTHOR_NAME" in
  *[Cc]ursor[Aa]gent*|*[Cc]ursor[- ][Aa]gent*|*[Cc]laude*|*[Cc]opilot*|*[Gg]ithub[- ][Cc]opilot*|*[Oo]pen[Aa][Ii]*)
    REAL_NAME=$(git config user.name)
    REAL_EMAIL=$(git config user.email)
    if [ -n "$REAL_NAME" ] && [ -n "$REAL_EMAIL" ]; then
      git commit --amend --no-edit --author="$REAL_NAME <$REAL_EMAIL>" --allow-empty >/dev/null 2>&1
    fi
    ;;
esac
```

## Common Pitfalls

1. Forgetting `chmod +x` — the hook silently does nothing if not executable
2. Using `sed -i ''` on Linux — this errors on GNU sed; omit the empty string
3. Not sharing the hooks — `.git/hooks/` is ignored by git, so teammates won't have them
4. Hook installed but wrong path — `git config core.hooksPath` may override `.git/hooks/`
5. Only installing `commit-msg` — this misses Cursor Agent which hijacks the **author** field, not the message body. Install both hooks.
6. Amending commits after the fact — the hook only runs on new commits, not `git commit --amend` in some edge cases (it does run for `--amend`)

## Best Practices

1. Install **both hooks** immediately at project creation, not after AI attribution has leaked in.
2. Use both layers: CLAUDE.md instructions (agent-level) + the hooks (git-level).
3. Commit `.githooks/` to the repo and configure `core.hooksPath` so hooks are shared automatically.
4. Run the verification tests after every new team member clones the repo.
5. Audit periodically: `git log --all --grep="Co-Authored-By" --oneline` and `git log --all --format="%an" | sort -u` to check if any attribution slipped through.

## Verification Checklist

Before calling setup complete:

1. `ls -la .git/hooks/commit-msg .git/hooks/post-commit` — both exist and are executable (`-rwxr-xr-x`).
2. Test commit with Co-Authored-By strips cleanly (see Workflow 2).
3. Test commit with AI author gets amended to your real identity (see Workflow 2).
4. `git log -1 --format="%B"` shows no `Co-Authored-By` line.
5. `git log -1 --format="%an"` shows your name, not an AI tool.
6. CLAUDE.md (if applicable) contains the no-attribution rule.
7. Team onboarding docs or `postinstall` script covers both hooks.
