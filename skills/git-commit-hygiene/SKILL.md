# Git Commit Hygiene

**Tier:** ESSENTIAL
**Category:** Engineering
**Domain:** Git / Version Control

## Overview

AI coding tools (Claude Code, Cursor, Copilot, etc.) often inject attribution lines into commit messages — `Co-Authored-By: Claude`, `Made-with: Cursor`, and similar. This skill installs a `commit-msg` git hook that silently strips all AI attribution before a commit is recorded, keeping your git history clean and human-authored.

Use this at project setup to enforce the policy permanently, without relying on agents remembering to omit attribution.

## Core Capabilities

- Strip all `Co-Authored-By:` lines (Claude Code, GitHub Copilot, etc.)
- Strip Cursor-specific attribution patterns
- Strip generic `Generated by ... AI` / `Created with ... AI` footers
- Remove trailing blank lines left behind after stripping
- Works on macOS and Linux
- Zero dependencies — pure shell

## When to Use

- Starting a new project where AI coding tools will be used
- A team member reports AI attribution appearing in git history
- Your CLAUDE.md or coding policy states no AI co-author lines
- You want to enforce commit hygiene without relying on per-agent configuration
- Adding `commit-msg` hook to an existing repo that lacks one

## Key Workflows

### 1. Install the Hook

Copy the hook script into the repo's `.git/hooks/` directory and make it executable:

```bash
cp .git/hooks/commit-msg.sample .git/hooks/commit-msg 2>/dev/null; true
curl -fsSL https://raw.githubusercontent.com/johnefemer/skillfish/main/skills/git-commit-hygiene/assets/commit-msg \
  -o .git/hooks/commit-msg
chmod +x .git/hooks/commit-msg
```

Or copy manually from `assets/commit-msg` in this skill directory.

### 2. Verify It Works

Stage any change and run a test commit with attribution:

```bash
git commit --allow-empty -m "test commit

Co-Authored-By: Claude Sonnet <noreply@anthropic.com>"
```

Then inspect the commit message:

```bash
git log -1 --format="%B"
```

Expected: only `test commit` — no `Co-Authored-By` line.

### 3. Add CLAUDE.md Rules (for Claude Code projects)

Add to your `CLAUDE.md`:

```markdown
## Git Commit Rules

- NEVER add `Co-Authored-By` lines to git commits — no AI co-author attribution
- Commit messages should be concise and human-style
```

This instructs the agent at the prompt level. The hook enforces it at the git level as a backstop.

### 4. Share the Hook With Your Team

Because `.git/hooks/` is not tracked by git, the hook must be re-installed by each developer. Options:

**Option A — Checked-in hooks dir:**

```bash
mkdir -p .githooks
cp .git/hooks/commit-msg .githooks/commit-msg
git config core.hooksPath .githooks
git add .githooks/commit-msg
git commit -m "Add commit-msg hook to strip AI attribution"
```

**Option B — Setup script:**

Add an `install-hooks.sh` to the repo root that developers run once after cloning:

```bash
#!/bin/sh
cp .githooks/commit-msg .git/hooks/commit-msg
chmod +x .git/hooks/commit-msg
echo "Git hooks installed."
```

**Option C — package.json postinstall:**

```json
{
  "scripts": {
    "postinstall": "cp .githooks/commit-msg .git/hooks/commit-msg && chmod +x .git/hooks/commit-msg"
  }
}
```

## Hook Script

The full hook (`assets/commit-msg`):

```sh
#!/bin/sh
# Strip ALL Co-Authored-By lines (Claude Code, Copilot, etc.)
sed -i '' '/^[[:space:]]*[Cc]o-[Aa]uthored-[Bb]y:/d' "$1"
# Strip Cursor attribution lines
sed -i '' '/^[[:space:]]*[Mm]ade-with:[[:space:]]*[Cc]ursor/d' "$1"
sed -i '' '/^[[:space:]]*[Gg]enerated by [Cc]ursor/d' "$1"
sed -i '' '/^[[:space:]]*[Cc]reated with [Cc]ursor/d' "$1"
sed -i '' '/[Cc]ursor AI/d' "$1"
# Strip any generic AI tool attribution
sed -i '' '/^[[:space:]]*[Gg]enerated by.*AI/d' "$1"
sed -i '' '/^[[:space:]]*[Cc]reated with.*AI/d' "$1"
# Remove trailing blank lines left behind
sed -i '' -e :a -e '/^\n*$/{$d;N;ba' -e '}' "$1"
```

> **macOS vs Linux:** The `sed -i ''` syntax is macOS/BSD. On Linux, use `sed -i` (no empty string argument). For a cross-platform hook, replace `sed -i ''` with `perl -i -pe` equivalents or detect the OS at the top of the script.

**Cross-platform variant:**

```sh
#!/bin/sh
# Detect sed flavor
if sed --version 2>/dev/null | grep -q GNU; then
  SED="sed -i"
else
  SED="sed -i ''"
fi

$SED '/^[[:space:]]*[Cc]o-[Aa]uthored-[Bb]y:/d' "$1"
$SED '/^[[:space:]]*[Mm]ade-with:[[:space:]]*[Cc]ursor/d' "$1"
$SED '/^[[:space:]]*[Gg]enerated by [Cc]ursor/d' "$1"
$SED '/^[[:space:]]*[Cc]reated with [Cc]ursor/d' "$1"
$SED '/[Cc]ursor AI/d' "$1"
$SED '/^[[:space:]]*[Gg]enerated by.*AI/d' "$1"
$SED '/^[[:space:]]*[Cc]reated with.*AI/d' "$1"
$SED -e :a -e '/^\n*$/{$d;N;ba' -e '}' "$1"
```

## Common Pitfalls

1. Forgetting `chmod +x` — the hook silently does nothing if not executable
2. Using `sed -i ''` on Linux — this errors on GNU sed; omit the empty string
3. Not sharing the hook — `.git/hooks/` is ignored by git, so teammates won't have it
4. Hook installed but wrong path — `git config core.hooksPath` may override `.git/hooks/`
5. Amending commits after the fact — the hook only runs on new commits, not `git commit --amend` in some edge cases (it does run for `--amend`)

## Best Practices

1. Install the hook immediately at project creation, not after AI attribution has leaked in.
2. Use both layers: CLAUDE.md instructions (agent-level) + the hook (git-level).
3. Commit `.githooks/` to the repo and configure `core.hooksPath` so the hook is shared automatically.
4. Run the verification test after every new team member clones the repo.
5. Audit periodically: `git log --all --grep="Co-Authored-By" --oneline` to check if any attribution slipped through.

## Verification Checklist

Before calling setup complete:

1. `ls -la .git/hooks/commit-msg` shows the file and is executable (`-rwxr-xr-x`).
2. Test commit with attribution strips cleanly (see Workflow 2 above).
3. `git log -1 --format="%B"` shows no `Co-Authored-By` line.
4. CLAUDE.md (if applicable) contains the no-attribution rule.
5. Team onboarding docs or `postinstall` script covers hook distribution.
